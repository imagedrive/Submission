メタデータ リンクの語彙

概要

語彙について

この語彙内のプロパティは、メタデータの link 要素における rel と properties 属性で利用可能である。

参照

プリフィックスのないリンク プロパティは、ベース IRI http://idpf.org/epub/vocab/package/link/# を使用して参照可能である。

リンクの関連性

次の値は、href 属性で参照されたリソースの関連性を確立するために link 要素の rel 属性で使用できる。

acquire

名前： acquire
概要： acquire キーワードは、何処で EPUB 出版物の完全なバージョンが入手できるのかを識別するために EPUB Previews と共に使用される。
個数： 0個以上
拡張： EPUB 出版物またはコレクションのみに適用。refines 属性が存在するとき、使用してはならない（MUST NOT）。
例： <link rel="acquire" href="http://example.org/book/9781448103706" media-type="text/html"/>

alternate

名前： alternate
概要：

alternate キーワードは、リンク用の HTML における alternate キーワードのサブセットである。それは次のように異なる。

  • 他のキーワードと組み合わせることができない。
  • 代替のリンクが、パッケージ ドキュメントのメタデータに包含されている場合、それは、media-type 属性に指定された形式で、パッケージ ドキュメントの代替表現を識別する。
  • 代替のリンクが、collection 要素のメタデータ [[!Packages]] に包含されている場合、それは、media-type 属性に指定された形式で、collection の代替表現を識別する。
  • リーディング システムは、代替リンクのハイパーリンクを生成する必要はない。
個数： 0個以上
拡張： EPUB 出版物またはコレクションのみに適用。refines 属性が存在するとき、使用してはならない（MUST NOT）。
例： <link rel="alternate" href="package.json" media-type="application/json-ld"/>

marc21xml-record（非推奨）

名前： marc21xml-record
概要： marc21xml-record キーワードの使用は、非推奨（deprecated）である。これは、media-type 属性 [[!Packages]] の値「application/marcxml+xml」を持つ record キーワードによって置き換えられた。
個数： 0 又は 1
拡張： EPUB 出版物またはコレクションのみに適用。refines 属性が存在するとき、使用してはならない（MUST NOT）。
例： <link rel="marc21xml-record" href="pub/meta/nor-wood-marc21.xml"/>

名前： mods-record
概要： mods-record キーワードの使用は、非推奨（deprecated）である。これは、media-type 属性 [[!Packages]] の値「application/mods+xml」を持つ record キーワードによって置き換えられた。
個数： 0 又は 1
拡張： EPUB 出版物またはコレクションのみに適用。refines 属性が存在するとき、使用してはならない（MUST NOT）。
例： <link rel="mods-record" href="pub/meta/nor-wood-mods.xml"/>

onix-record（非推奨）

名前： onix-record
概要： onix-record キーワードの使用は、非推奨（deprecated）である。これは、プロパティの属性 [[!Packages]] の値 onix を持つ record キーワードによって置き換えられた。
個数： 0 又は 1
拡張： EPUB 出版物またはコレクションのみに適用。refines 属性が存在するとき、使用してはならない（MUST NOT）。
例： <link rel="onix-record" href="pub/meta/nor-wood-onix.xml"/>

record

名前： record
概要：

参照されたリソースがメタデータ レコードであることを示す。

このキーワードが特定されている時、レコードのメディア タイプは、media-type 属性 [[!Packages]] で識別される。

通常のリンクされたメタデータ レコードについては、EPUB Linked Metadata Guide を参照されたい。

レコードの形式が、メディア タイプから識別できない場合、識別子のプロパティは、properties 属性 [[!Packages]] で割り当てることができる。
個数： 0個以上
拡張： EPUB 出版物またはコレクションのみに適用。refines 属性が存在するとき、使用してはならない（MUST NOT）。
例： <link rel="record" href="book/52.atom" media-type="application/atom+xml;type=entry;profile=opds-catalog"/>

xml-signature（非推奨）

名前： xml-signature
概要： xml-signature キーワードの使用は、非推奨（deprecated）である。これは、他のリンク方法によって置き換えられない。XML signatures の識別は、EPUB の将来のバージョンで対処されるだろう。
個数： 0個以上
拡張： 全てのプロパティ。
例： <link refines="#meta-authority-01" rel="xml-signature" href="../META-INF/signatures.xml#MAI-Signature"/>

xmp-record（非推奨）

名前： xmp-record
概要： xmp-record キーワードの使用は、非推奨（deprecated）である。これは、プロパティ属性の値 xmp を持つ record キーワードによって置き換えられた。
個数： 0 又は 1
拡張： EPUB 出版物またはコレクションのみに適用。refines 属性が存在するとき、使用してはならない（MUST NOT）。
例： <link rel="xmp-record" href="pub/meta/nor-wood-xmp.xml"/>

リンクのプロパティ

次の値は、参照されたリソースが表すレコードの形式を確立するために、link 要素の properties 属性で使用できる。これらの値は、メディア タイプによって一意に識別できないレコードの形式のために提供される。

onix

名前： onix
概要： onix プロパティは、参照されたリソースが、ONIX レコード [[!ONIX]] であることを示す。
例： <link rel="record" href="pub/meta/nor-wood-onix.xml" media-type="application/xml" properties="onix"/>

xmp

名前： xmp
概要： xmp プロパティは、参照されたリソースが、XMP レコード [[!XMP]] であることを示す。
例： <link rel="record" href="pub/meta/nor-wood-xmp.xml" media-type="application/xml" properties="xmp"/>
