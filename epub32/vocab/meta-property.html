メタ プロパティの語彙

概要

語彙について

この語彙のプロパティは、meta 要素の property 属性で使用できる。

参照

プリフィックスのないプロパティは、ベース IRI http://idpf.org/epub/vocab/package/meta/# を使用して参照可能である。

メタデータ meta プロパティ

次の表は、meta 要素の property 属性で使用するプロパティを定義する。

「拡張」フィールドで特に指示がない限り、のこのセクションで定義されたpropertiesは、部分式（subexpression）を定義するために使用される。言い換えると、このセクションで定義されているプロパティを伝える meta 要素は、拡張されたリソース又は表現を参照する refines 属性を持たなければならない（）。

各プロパティの定義で、許容された値フィールドは、（[[!XMLSCHEMA-2]] データタイプを使用する）値の期待された型を示し、個数フィールドは、プロパティを他のプロパティに紐づけても良い（MAY）回数を示し、拡張できる対象フィールドは、紐づけても良い（MAY）プロパティを示す。

alternate-script

名前： alternate-script
概要：

alternate-script プロパティは、xml:lang 属性によって識別される言語とスクリプト内の関連するプロパティ値の代替表現を提供する。

このプロパティは、国際化目的のために creator と title プロパティに、典型的に付けられる。
許可された値： xsd:string
個数： 0個以上
拡張できる対象： すべてのプロパティ

次の例は、英語の著者名及び日本語の代替表現を示している。

<metadata xmlns:dc="http://purl.org/dc/elements/1.1/">
    …
    <dc:creator id="creator">Haruki Murakami</dc:creator>
    <meta refines="#creator" property="alternate-script" xml:lang="ja">村上 春樹</meta>
    …
</metadata>

authority

名前： authority
概要：

authority プロパティは、参照された要素の値が引き出されるシステム又はスキームを示している。
許可された値：

  • reserved authority value [[!AuthorityRegistry]] （大文字と小文字を区別しない）、または、

  • スキームを識別する絶対 IRI [[!RFC3987]]。IRI は、スキームに関する詳細情報を提供するリソースを参照するべきである（SHOULD）。
個数： 0個以上
拡張できる対象： subject

次の例は、BISAC サブジェクトの見出しを示している。

<metadata xmlns:dc="http://purl.org/dc/elements/1.1/">
    …
    <dc:subject id="subject01">FICTION / Occult &amp; Supernatural</dc:subject>
    <meta refines="#subject01" property="authority">BISAC</meta>
    …
</metadata>

belongs-to-collection

名前： belongs-to-collection
概要：

belongs-to-collection プロパティは、EPUB 出版物が属するコレクションの名前を識別する。EPUB 出版物は、ひとつ以上のコレクションを識別しても良い（MAY）。

あるコレクションが、他のコレクションのメンバーであることを識別するために refines 属性を使用してそれらのプロパティを連鎖させることも可能である。

リーディング システムが、コレクションを体系化し、（例えば、無関係なコレクションが、似たような名前を共有するかもしれないし、コレクションの異なるエディションがリリースされるかもしれないなど）名前の衝突を避けるために、識別子は、コレクションのインスタンスに一意な識別子を提供するべきである（SHOULD）。dcterms:identifier プロパティは、この識別子を伝えなければならない。

コレクションは、collection-type プロパティを紐付けることによって、その性質をより詳細に定義しても良い（MAY）。

コレクション内にある EPUB 出版物の位置は、group-position プロパティを紐付けることによって、提供しても良い（MAY）。
許可された値： xsd:string
個数： 0個以上
拡張できる対象： EPUB 出版物に適用され、それ自体の他のインスタンスを改良することができる。

次の例は、出版物が、ハリー ポッターの書籍セットに属していることを示している。

<metadata>
    …
    <meta property="belongs-to-collection" id="c02">Harry Potter</meta>
    <meta refines="#c02" property="collection-type">set</meta>
    …
</metadata>

collection-type

名前： collection-type
概要：

collection-type プロパティは、コレクションの形式又は性質を示している。

collection-type 値が、コードのリスト又は他の形式の一覧表から引き出される時、scheme 属性は、そのソースを識別するために紐付けるべきである（SHOULD）。

スキームが特定されていない時、リーディング システムは、次のコレクション型の値を認識するべきである（SHOULD）。

series

グループとして形式的に識別された一連の関連作品で、典型的に上限がなく、時間の経過と共に個別に発行される。

set

ひとつの知的なユニットを同時に構成する有限の作品コレクションで、典型的に同時に発行され、ユニットとして販売できる。
許可された値： xsd:string
個数： 0 又は 1
拡張できる対象： belongs-to-collection

次の例は、出版物が、The New French Cuisine Masters シリーズに属することを示している。

<metadata>
    …
    <meta property="belongs-to-collection" id="c01">
        The New French Cuisine Masters
    </meta>
    <meta refines="#c01" property="collection-type">series</meta>
    …
</metadata>

display-seq (Deprecated)

名前： display-seq
概要：

display-seq は、（例えば、複数の title をレンダリングする順番を識別するためになど）同じメタデータ プロパティに関連する現在のプロパティを表示する数値的な位置を識別する。

display-seq プロパティの使用は、非推奨（deprecated）である。優先順位は、EPUB パッケージ ドキュメントのドキュメント順に出現順に基づいて、セット内の要素に与えられる（すなわち、ドキュメント順の最初の要素は、最も優先度が高い）。
許可された値： xsd:unsignedInt
個数： 0 又は 1
拡張できる対象： 全てのプロパティ

file-as

名前： file-as
概要： file-as プロパティは、ソートに関連するプロパティの正規化された形式を提供する。
許可された値： xsd:string
個数： 0 又は 1
拡張できる対象： 全てのプロパティ 

<metadata xmlns:dc="http://purl.org/dc/elements/1.1/">
    …
    <dc:creator id="creator01">Lewis Carroll</dc:creator>
    <meta refines="#creator01" property="file-as">Carroll, Lewis</meta>
    …
</metadata>

group-position

名前： group-position
概要：

group-position プロパティは、EPUB 出版物が、（全て EPUB 出版物かそれともそうでないか）同じグループに属する他の作品と比較して順序づけられる数値的な位置を識別する。

group-position プロパティは、グループを確立する任意のメタデータ プロパティに紐付けることができるが、典型的に、belongs-to-collection プロパティに関連付けられる。

EPUB 出版物は、複数のグループに属することができる。
許可された値： ひとつの xsd:unsignedInt 又は少数で区切られた一連の数字（例：1 や 2.2.1）。
個数： 0 又は 1
拡張できる対象： 全てのプロパティ

次の例は、出版物が、指輪物語のセットの2番目であることを示している。

<metadata>
    …
    <meta property="belongs-to-collection" id="c01">
        The Lord of the Rings
    </meta>
    <meta refines="#c01" property="collection-type">set</meta>
    <meta refines="#c01" property="group-position">2</meta>
    …
</metadata>

The following example uses a decimal-separated value for group-position, in this case volume 98, issue 4 of a periodical.

<metadata>
    …
    <meta property="belongs-to-collection" id="cygnus-x-1">Physical Review D</meta>
    <meta refines="#cygnus-x-1" property="collection-type">series</meta>
    <meta refines="#cygnus-x-1" property="group-position">98.4</meta>
    …
</metadata>

identifier-type

名前： identifier-type
概要：

identifier-type プロパティは、identifier の形式又は性質を示す。

identifier-type 値が、コードのリスト又は他の形式の一覧表から引き出される時、scheme 属性は、そのソースを識別するために紐付けるべきである（SHOULD）。
許可された値： xsd:string
個数： 0 又は 1
拡張できる対象： identifier、source

アクセシブルなページ ナビゲーションを提供する方法についての情報は [[EPUBAccessibilityTechniques]] を見よ。

次の例は、識別子が、ONIX コード リスト 5 の数値を使用する ISBN 示す方法の例である。

<metadata xmlns:dc="http://purl.org/dc/elements/1.1/">
    …
    <dc:identifier id="isbn-id">urn:isbn:9780101010101</dc:identifier>
    <meta refines="#isbn-id" property="identifier-type" scheme="onix:codelist5">15</meta>
    …
</metadata>

meta-auth（非推奨）

名前： meta-auth
概要：

meta-auth プロパティは、パッケージ メタデータのインスタンスに責任のある、関係者又は権限を識別する。

meta-auth プロパティの仕様は、非推奨（deprecated）である。

許可された値： xsd:string
個数： 0 又は 1
拡張できる対象： 全てのプロパティ

role

名前： role
概要：

role プロパティは、（例えば、人物が、作品の著者や編集者など）creator 又は contributor によって行われる作品の性質を説明する。

role 値が、コードのリスト又は他の形式の一覧表から引き出される時、scheme 属性は、そのソースを識別するために紐付けるべきである（SHOULD）。
許可された値： xsd:string
個数： 0 又は 1
拡張できる対象： contributor、creator

次の例は、作品の著者とイラストレーターを区別するために、MARC Relators vocabulary を使用している。

<metadata xmlns:dc="http://purl.org/dc/elements/1.1/">
    …
    <dc:creator id="creator01">Lewis Carroll</dc:creator>
    <meta refines="#creator01" property="role" scheme="marc:relators">aut</meta>
    
    <dc:creator id="creator02">John Tenniel</dc:creator>
    <meta refines="#creator02" property="role" scheme="marc:relators">ill</meta>
    …
</metadata>

source-of

名前： source-of
概要：

source-of プロパティは、EPUB 出版物の特定のレンディションが保持している、適合するソースのリソースの固有の面を示す。

この仕様は、参照される dc:source 要素が、コンテンツに定義された pagebreak プロパティのソースであることを示す pagination 値を定義する。この値は、ページネーションが包含される及び底本が分かっているときはいつでも、設定されるべきである（SHOULD）。
許可された値： pagination
個数： 0 又は 1
拡張できる対象： dc:source

次の例は、派生元になる底本のソース ISBN 識別子と一緒に EPUB 出版物の ISBN 識別子を示している。

<metadata xmlns:dc="http://purl.org/dc/elements/1.1/">
    …
    <dc:identifier id="isbn-id">urn:isbn:9780101010101</dc:identifier>
    <meta refines="#isbn-id" property="identifier-type" scheme="onix:codelist5">15</meta>
    
    <dc:source id="src-id">urn:isbn:9780375704024</dc:source>
    <meta refines="#src-id" property="identifier-type" scheme="onix:codelist5">15</meta>
    <meta refines="#src-id" property="source-of">pagination</meta>
    …
</metadata>

term

名前： term
概要：

term プロパティは、主題のコードを提供する。
許可された値： xsd:string
個数： 0 又は 1
拡張できる対象： subject

次の例は、主題の見出しの BISAC コードを示している。

<metadata xmlns:dc="http://purl.org/dc/elements/1.1/">
    …
    <dc:subject id="subject01">FICTION / Occult &amp; Supernatural</dc:subject>
    <meta refines="#subject01" property="authority">BISAC</meta>
    <meta refines="#subject01" property="term">FIC024000</meta>
    …
</metadata>

title-type

名前： title-type
概要：

title-type プロパティは、title の形式又は性質を示す。

title-type 値が、コードのリスト又は他の形式の一覧表から引き出される時、scheme 属性は、そのソースを識別するために紐付けるべきである（SHOULD）。スキームが特定されていない時、リーディング システムは、次のタイトル型の値を認識するべきである（should）：main、subtitle、short、collection、edition 及び expanded。
許可された値： xsd:string
個数： zero or one
拡張できる対象： title

次の例は、異なるタイトル型を識別する方法を示している。

<metadata xmlns:dc="http://purl.org/dc/elements/1.1/">
    …
    <dc:title id="t1">A Dictionary of Modern English Usage</dc:title>
    <meta refines="#t1" property="title-type">main</meta>
    
    <dc:title id="t2">First Edition</dc:title>
    <meta refines="#t2" property="title-type">edition</meta>
    
    <dc:title id="t3">Fowler's</dc:title>
    <meta refines="#t3" property="title-type">short</meta>
    …
</metadata>

次の例は、複合的なタイトルの「The Great Cookbooks of the World: Mon premier guide de cuisson, un Mémoire. The New French Cuisine Masters, Volume Two. Special Anniversary Edition」を分類する方法を示している。

<metadata xmlns:dc="http://purl.org/dc/elements/1.1/">
    <dc:title id="t1" xml:lang="fr">Mon premier guide de cuisson, un Mémoire</dc:title>
    <meta refines="#t1" property="title-type">main</meta>
    <meta refines="#t1" property="display-seq">2</meta>
    
    <dc:title id="t2">The Great Cookbooks of the World</dc:title>
    <meta refines="#t2" property="title-type">collection</meta>
    <meta refines="#t2" property="display-seq">1</meta>
    
    <dc:title id="t3">The New French Cuisine Masters</dc:title>
    <meta refines="#t3" property="title-type">collection</meta>
    <meta refines="#t3" property="display-seq">3</meta>
    
    <dc:title id="t4">Special Anniversary Edition</dc:title>
    <meta refines="#t4" property="title-type">edition</meta>
    <meta refines="#t4" property="display-seq">4</meta>
    
    <dc:title id="t5">The Great Cookbooks of the World: 
        Mon premier guide de cuisson, un Mémoire. 
        The New French Cuisine Masters, Volume Two. 
        Special Anniversary Edition</dc:title>
    <meta refines="#t5" property="title-type">expanded</meta>
    …
</metadata>

例

次の例は、レンディションに含まれる洗練されたメタデータの典型的なセットを表している。

<metadata xmlns:dc="http://purl.org/dc/elements/1.1/">
    
    <dc:identifier id="pub-id">urn:uuid:A1B0D67E-2E81-4DF5-9E67-A64CBE366809</dc:identifier>
    <meta refines="#pub-id" property="identifier-type" scheme="xsd:string">uuid</meta>
    
    <dc:identifier id="isbn-id">urn:isbn:9780101010101</dc:identifier>
    <meta refines="#isbn-id" property="identifier-type" scheme="onix:codelist5">15</meta>
    
    <dc:source id="src-id">urn:isbn:9780375704024</dc:source>
    <meta refines="#src-id" property="identifier-type" scheme="onix:codelist5">15</meta>
    
    <dc:title id="title">Norwegian Wood</dc:title>
    <meta refines="#title" property="title-type">main</meta>
    
    <dc:language>en</dc:language>
    
    <dc:creator id="creator">Haruki Murakami</dc:creator>
    <meta refines="#creator" property="role" scheme="marc:relators" id="role">aut</meta>
    <meta refines="#creator" property="alternate-script" xml:lang="ja">村上 春樹</meta>
    <meta refines="#creator" property="file-as">Murakami, Haruki</meta>
    
    <meta property="dcterms:modified">2011-01-01T12:00:00Z</meta>
    
</metadata>
